Partial Least Squares
Model / Proxy Supervised Dimension Reduction Derived Inputs
Like PCR, but the directions are chosen using the response y — find linear combinations of X that have high variance AND high correlation with y.
Benefits & Tradeoffs
What does PLS buy you over PCR and OLS?
Benefits
  • Supervised direction-finding: unlike PCR, PLS uses y when constructing the derived inputs. It seeks directions that have both high variance in X and high correlation with y. (ESL §3.5.2)
  • Can find signal that lives in low-variance directions of X — directions that PCR would discard. If y depends on a direction with small eigenvalue, PCR misses it; PLS can still find it. (ESL §3.5.2)
  • Variance reduction: like PCR, using \(M < p\) directions reduces the variance of the fitted coefficients relative to OLS.
  • At \(M = p\), recovers the full OLS solution — nests OLS as a special case. (ESL §3.5.2)
Tradeoffs
  • Can increase variance: because PLS uses y in constructing directions, it introduces additional randomness. ESL notes that while PLS can decrease bias, it can also increase variance — so the net benefit is not guaranteed. (ESL §3.5.2)
  • Not scale invariant: like PCR, results depend on predictor scaling. Inputs must be standardized. (ESL §3.5.2)
  • In practice, PLS often does not outperform ridge regression. ESL is candid: the supervised dimension selection does not always compensate for the added variance. (ESL §3.5.2)
  • Less interpretable than PCR — the PLS directions depend on y, so they change when y changes. The "meaning" of each component is less stable.
"Partial least squares seeks directions that have high variance and have high correlation with the response, in contrast to principal components regression which keys only on high variance." — ESL §3.5.2

Visualization 2: How PLS Builds Directions (NIPALS)
PLS constructs its directions iteratively. The first direction \(\hat{\phi}_1\) is simply \(X^\top y\) (normalized). After projecting out that direction, it repeats on the residuals. Compare this to PCA, which ignores y entirely.

PCR (Unsupervised)

  • Compute SVD of X: \(X = UDV^\top\)
  • Take first M columns of \(Z = UD\)
  • y is never consulted during direction selection
  • Directions maximize \(\text{Var}(X\phi)\)

PLS (Supervised)

  • Direction 1: \(\hat{\phi}_1 \propto X^\top y\) (univariate regression weights)
  • Score: \(z_1 = X\hat{\phi}_1\)
  • Orthogonalize X w.r.t. \(z_1\), repeat
  • Directions maximize \(\text{Cov}(X\phi, y)^2\)
1

At m=1, PLS picks the direction most correlated with y. At m=2, it deflates X (removes the first direction's contribution) and picks the next best direction from the residual. By m=p, PLS and OLS coincide.

All three methods can be understood through their shrinkage of the principal component coefficients. Ridge shrinks smoothly. PCR hard-thresholds. PLS does something in between — it tends to shrink the low-variance directions but can also inflate some directions.

PLS shrinkage factors are data-dependent and can exceed 1 (inflation) for some components. This is because PLS uses y to weight directions, which can amplify certain components beyond their OLS value. This inflation is the source of PLS's increased variance.

Statistical Guarantees
Formal properties of the PLS estimator
PropertyStatusDetails
Unbiased No Biased for \(M < p\); only unbiased when \(M = p\) (recovers OLS)
Lower Variance than OLS Yes Using \(M < p\) directions generally reduces variance relative to OLS
Better MSE than PCR Not guaranteed PLS can decrease bias (supervised directions) but increases variance (uses y); net effect is data-dependent
Orthogonal Latent Directions Yes PLS directions are constructed to be orthogonal
Orthogonal Scores Yes The derived scores \(z_1, \ldots, z_M\) are orthogonal
Nests OLS Yes At \(M = p\), the PLS fit equals the OLS fit
Scale Invariant No Results depend on predictor scaling; standardize first
Consistent Conditional Consistent when M grows with n at an appropriate rate
Formula Reference
For lookup after understanding the concept
PLS direction (NIPALS Algorithm): \[ \hat{\phi}_{1j} = \langle x_j, y \rangle \quad \text{(univariate regression coefficients)} \] \[ z_1 = \sum_j \hat{\phi}_{1j} x_j = X\hat{\phi}_1 \]
The first PLS direction weights each predictor by its univariate correlation with y. This is the key difference from PCA, which ignores y.
Subsequent directions (deflation): \[ \text{Orthogonalize } x_1, \ldots, x_p \text{ with respect to } z_1 \] \[ \hat{\phi}_{2j} = \langle x_j^{(1)}, y \rangle \quad \text{where } x_j^{(1)} \text{ is the orthogonalized } x_j \]
After extracting the first direction, PLS deflates X (removes z_1's contribution) and repeats. This iterative process is the NIPALS algorithm.
Optimization form: \[ \hat{\phi}_m = \arg\max_{\phi} \; \text{Cov}(X\phi, y)^2 \quad \text{subject to } \|\phi\| = 1, \; \phi \perp \hat{\phi}_1, \ldots, \hat{\phi}_{m-1} \]
PLS can be seen as sequential maximization of squared covariance between the derived direction and y, subject to orthogonality constraints. Contrast with PCA: \(\arg\max_\phi \text{Var}(X\phi)\).
PLS fit: \[ \hat{y}^{\text{pls}}_M = \bar{y}\mathbf{1} + \sum_{m=1}^{M} \hat{\theta}_m z_m \]
Same form as PCR, but with PLS-derived directions \(z_m\) instead of principal components.
Algorithm (NIPALS)
Step-by-step procedure for M PLS components
#Step
0Standardize each \(x_j\) to mean 0, variance 1. Set \(\hat{y}^{(0)} = \bar{y}\mathbf{1}\), \(x_j^{(0)} = x_j\)
1For \(m = 1, 2, \ldots, M\):
1a\(\hat{\phi}_{mj} = \langle x_j^{(m-1)}, y \rangle\) for each \(j = 1, \ldots, p\)
1b\(z_m = \sum_j \hat{\phi}_{mj} x_j^{(m-1)}\) (the m-th PLS direction)
1c\(\hat{\theta}_m = \langle z_m, y \rangle / \langle z_m, z_m \rangle\) (regress y on \(z_m\))
1dOrthogonalize each \(x_j^{(m-1)}\) with respect to \(z_m\): \(x_j^{(m)} = x_j^{(m-1)} - [\langle z_m, x_j^{(m-1)}\rangle / \langle z_m, z_m \rangle] z_m\)
2Output: \(\hat{y}^{\text{pls}}_M = \bar{y}\mathbf{1} + \sum_{m=1}^M \hat{\theta}_m z_m\)
