model / proxy § 3.5.1 — Principal Components Regression

Principal Components Regression

\[ \hat{y}^{\text{pcr}}_M = \bar{y}\mathbf{1} + \sum_{m=1}^{M} \hat{\theta}_m z_m \]
where \(z_m = Xv_m\) are the derived principal component columns, \(\hat{\theta}_m = \langle z_m, y \rangle / \langle z_m, z_m \rangle\)

PCR hard-thresholds: components with index > M are completely dropped. Contrast with ridge, which applies continuous shrinkage based on eigenvalue magnitude.

Both methods shrink via eigenvalues of XᵀX. Ridge: factor \(d_j^2/(d_j^2+\lambda)\). PCR: 1 if kept, 0 if dropped.

As M increases, bias decreases (more signal captured) but variance increases (more directions, potentially noisy). Optimal M balances these.

Key Facts

"Principal component regression forms the derived input columns z_m = Xv_m, and then regresses y on z_1, z_2, ..., z_M for some M ≤ p."
  • Uses SVD of X: X = UDVᵀ, principal components are columns of UD
  • Discards low-variance directions (small eigenvalues of XᵀX)
  • M is a tuning parameter chosen by cross-validation
  • Shrinkage is discrete (hard thresholding), not continuous

Guarantees / Properties

Biased Unbiased Biased when M < p; unbiased only if M = p (= OLS)
Lower Var Variance Reduction Drops high-variance directions from low-eigenvalue components
Conditional Consistent Consistent when M grows with n appropriately
Hard Smooth Shrinkage Hard 0/1 threshold — not smooth like ridge

Algorithm Steps

  • 1. Compute SVD of centered X: X = UDVᵀ
  • 2. Form principal components z_m = Xv_m (columns of UD)
  • 3. Choose M via cross-validation
  • 4. OLS regression of y on z_1, ..., z_M
  • 5. Map coefficients back to original space
