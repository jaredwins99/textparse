Principal Components Regression
Model / Proxy Dimension Reduction Derived Inputs
Reduce the predictor space to its highest-variance directions, then run ordinary regression in that lower-dimensional space.
Benefits & Tradeoffs
What does PCR buy you? What do you lose?
Benefits
  • Variance reduction by discarding low-eigenvalue directions — the small principal components are those most affected by collinearity and contribute most to variance in OLS estimates. (ESL §3.5.1)
  • Avoids collinearity entirely: the retained principal components are orthogonal by construction, so the regression in the reduced space has no multicollinearity. (ESL §3.5.1)
  • When \(M = p\), recovers the full OLS solution — PCR nests OLS as a special case, so you never lose expressive power if you choose to keep all components. (ESL §3.5.1)
  • Works well when the response depends primarily on directions of high variance in X — which is common when predictors are correlated measurements of a few underlying factors.
Tradeoffs
  • Unsupervised dimension reduction — principal components are chosen based on variance of X alone, ignoring the response y. A high-variance direction of X may be irrelevant to y. (ESL §3.5.1)
  • Hard thresholding: each component is either fully kept or completely dropped. This is a discrete, non-smooth form of shrinkage. Ridge regression applies continuous shrinkage to the same components. (ESL §3.5.1)
  • Not scale invariant: results depend on the scaling of the predictors, so inputs must be standardized before computing components. (ESL §3.5.1)
  • Coefficient interpretation is lost — because the regression is in the principal component space, the relationship between original predictors and the response is indirect.
"Principal component regression forms the derived input columns \(z_m = Xv_m\), and then regresses y on \(z_1, z_2, \ldots, z_M\) for some \(M \le p\). Since the \(z_m\) are orthogonal, this regression is just a sum of univariate regressions." — ESL §3.5.1

Both methods operate on the same eigenvalue decomposition of \(X^\top X\). Ridge applies smooth shrinkage: factor \(d_j^2/(d_j^2 + \lambda)\). PCR applies binary shrinkage: 1 if kept, 0 if dropped. Ridge is "softer" but PCR is more interpretable (you literally pick how many components to use).

Visualization 3: Bias-Variance Tradeoff
As M increases, bias falls (more signal captured) but variance rises (noisier directions included). The optimal M balances these.

At \(M = p\) (all components), PCR equals OLS — zero bias from dimension reduction, but maximum variance. At small M, you get aggressive variance reduction at the cost of potentially missing signal that lives in the dropped components.

Statistical Guarantees
Formal properties of the PCR estimator
PropertyStatusDetails
Unbiased M = p only Biased when \(M < p\); unbiased only when \(M = p\) (recovers OLS)
Variance Reduction Yes Drops high-variance directions from low-eigenvalue components
Consistency Conditional Consistent when M grows with n at an appropriate rate
Smooth Shrinkage No Hard 0/1 threshold on eigenvalue components — not continuous like ridge
Scale Invariant No Results depend on predictor scaling; standardize inputs first
Nests OLS Yes At \(M = p\), the PCR fit is identical to the OLS fit
Formula Reference
For lookup after understanding the concept
\[ \hat{y}^{\text{pcr}}_M = \bar{y}\mathbf{1} + \sum_{m=1}^{M} \hat{\theta}_m z_m \]
where \(z_m = Xv_m\) are the principal component columns derived from the SVD \(X = UDV^\top\), and \(\hat{\theta}_m = \langle z_m, y \rangle / \langle z_m, z_m \rangle\) is the univariate regression coefficient on each component.
\[ \hat{y}^{\text{pcr}}_M = \sum_{m=1}^{M} u_m u_m^\top y = H_M y \]
Equivalently, PCR projects y onto the subspace spanned by the first M columns of U (the left singular vectors of X). The hat matrix \(H_M = \sum_{m=1}^M u_m u_m^\top\) is an orthogonal projection.
Shrinkage factors (vs ridge):
PCR: \(s_j = \mathbf{1}[j \le M]\)   —   Ridge: \(s_j = \frac{d_j^2}{d_j^2 + \lambda}\)
Both methods shrink via eigenvalues of \(X^\top X\), but PCR applies discrete keep/drop while ridge applies continuous damping.
Algorithm
Step-by-step procedure
#Step
1Standardize each column of X to mean 0, variance 1
2Compute SVD of centered X: \(X = UDV^\top\)
3Form principal components: \(z_m = Xv_m = u_m d_m\)
4Choose M (number of components) by cross-validation
5Regress y on \(z_1, \ldots, z_M\) by OLS (since orthogonal, this is M univariate regressions)
6Map coefficients back to original space: \(\hat{\beta}^{\text{pcr}} = \sum_{m=1}^M \hat{\theta}_m v_m\)
