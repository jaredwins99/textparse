L1 Arc Length (Total Variation)

ESL §3.4.4 Path complexity measure

Benefits & Tradeoffs

What it buys you

A scale-free way to parametrize solution paths: instead of using lambda (which is problem-specific), arc length measures how far the coefficients have actually traveled.
Additive across predictors -- you can decompose the total path complexity into per-variable contributions and see which coefficients are "moving the most."
For monotone paths (LAR), arc length equals the L1 norm of the current coefficient vector, making it a natural measure of model complexity.

Limitations

Not monotone for lasso paths with sign changes -- when a variable exits and re-enters, arc length keeps accumulating but the coefficient backtracks.
Purely a descriptive measure of path geometry -- it does not directly inform model selection or prediction quality.
"The L1 arc length of LAR is simply the sum of the absolute values of the coefficient changes at each step." -- ESL §3.4.4, p. 76

Reference: Definition

\(\displaystyle \mathrm{TV}(\hat\beta, S) = \int_0^S \|\dot{\hat\beta}(s)\|_1\, ds = \sum_{j=1}^{p} \mathrm{TV}(\hat\beta_j, S)\)
\(\text{For piecewise linear paths: } \mathrm{TV} = \sum_j \sum_k |\Delta\hat\beta_j^{(k)}|\)

The total variation reduces to a sum of absolute coefficient changes at each breakpoint. For monotone paths (like LAR), TV equals the total increase in L1 norm: TV = ||beta_hat||_1. For non-monotone lasso paths, TV >= ||beta_hat||_1 because backtracking accumulates additional arc length.
