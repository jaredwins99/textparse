Piecewise-Linear Solution Path

ESL §3.4.4 Efron et al. 2004 LAR / Lasso / Homotopy

Benefits & Tradeoffs

Why this matters

The entire lasso solution path can be computed exactly with the same cost as a single least squares fit -- no need to solve on a grid of lambda values.
Between breakpoints, the active set is constant and coefficients move in a fixed direction. This means you get the solution at every possible lambda, not just a discrete set.
At most min(N, p) breakpoints -- the path has finite complexity regardless of the penalty range.

Limitations

Only holds for penalties that produce piecewise-linear KKT conditions (L1-type). Ridge, elastic net (for the L2 part), and non-convex penalties produce smooth or non-linear paths.
In practice, pathwise coordinate descent on a dense lambda grid is often faster than the exact LAR/homotopy algorithm for large p, despite losing the exact-path guarantee.
"By construction the coefficients in LAR change in a piecewise linear fashion." -- ESL §3.4.4, p. 74

Reference: Path Structure

\(\displaystyle \hat\beta(\lambda) = \hat\beta(\lambda_k) + (\lambda - \lambda_k)\, \mathbf{v}_k, \qquad \lambda \in [\lambda_{k+1},\, \lambda_k]\)
\(\text{where } \mathbf{v}_k \text{ is determined by } X_{\mathcal{A}_k} \text{ and the active set } \mathcal{A}_k \text{ is constant on each segment}\)

The direction v_k on each segment comes from solving the KKT conditions with the current active set. At a breakpoint, one KKT condition becomes tight (a new variable's correlation reaches the maximum) or one coefficient crosses zero. Either event changes the active set and thus changes v_k, producing the kink.
